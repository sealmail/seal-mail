基础必须完好

首先是一个称手的邮件客户端

信任层建立在扎实的基础之上：多协议、多账户、离线可用，日常收发不打折扣。

多账户 · 多协议

IMAP / POP3 收件 + SMTP 发件（SSL / STARTTLS）。预设 Exchange Online（OAuth2 设备码登录）、自建 Exchange、Gmail、iCloud、QQ、163，或自定义服务器。

本地缓存 · 离线阅读

SQLite 本地缓存 + 增量同步，离线照常阅读；IMAP IDLE / POP3 轮询实时通知新邮件，历史邮件分页加载。

过滤规则 · 自建目录

按发件人 / 主题 / 正文匹配，自动归档到目录，可一键「立即整理收件箱」。IMAP 建真实服务器目录，POP3 用本地虚拟目录。

安全渲染

HTML 正文安全渲染，默认阻止远程图片追踪；删除先进回收站，撤销发送给你反悔的时间。

完整收发体验

统一收件箱、会话线程、星标归档、本地搜索、附件收发、本地草稿、收件人自动补全与常用快捷键。

高风险拦截

启发式识别付款催促、索取密码 / 助记词、合同时限等高风险邮件，弹窗要求「已独立核实」才能继续操作。

信任层 · 特色功能

像火漆封印一样，为邮件盖上可验证的印

不依赖头像、签名档装饰或语言习惯——只认密钥指纹。

01

发送时盖印

二选一：本地生成的 Ed25519 密钥（identity.key，0600），或 Ledger 硬件密钥——每次签名都在设备上确认，私钥永不离开硬件。

02

对普通收件人隐形

签名放在 X-SealMail-* 邮件头中，正文只追加一行标准 --  签名档式的低调说明——在 Gmail、Outlook 里看起来就是一封普通邮件。

03

接收时本地验证

验证全部在本机完成，不经过任何服务器。五种封印状态一眼分辨：本人、可信、未知、被篡改、被冒充。
